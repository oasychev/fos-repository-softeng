# Методические рекомендации для преподавателя по использованию учебных и практических ресурсов Модуля 3
## Тема: «Продвинутые операции с системой контроля версий Git»


## 1. Общие положения

Ресурсы модуля разделены на две категории: **Учебные материалы** (теоретическая база) и **Практические ресурсы** (архивы репозиториев и ссылки на Open Source проекты). Преподавателю рекомендуется использовать их не как пассивный справочник, а как активный инструмент для:
- нивелирования разрыва в базовой подготовке студентов;
- подготовки демонстрационных материалов перед выполнением КИМ;
- предоставления эталонных примеров при разборе ошибок.

## 3. Методика использования практических ресурсов (Репозитории)

Эти ресурсы предназначены в первую очередь для обеспечения КИМ-3.2 (Лабораторная работа). Хотя по требованиям студенты готовят репозитории самостоятельно, эти архивы служат критически важным страховочным и демонстрационным инструментом.

| Наименование ресурса | Целевое применение в учебном процессе | Рекомендации преподавателю |
| :--- | :--- | :--- |
| **M3 repo1 branches-without-conflicts.zip** | Демонстрация, резервный вариант для КИМ-3.2 (Часть 1). | Использовать для показа *идеального* сценария `cherry-pick` или `rebase` без конфликтов. Если студент принес некорректно подготовленный собственный репозиторий, выдать ему этот архив в качестве замены, со снижением балла за этап подготовки. |
| **M3 repo2 branches with conflicts.zip** | Эталон сложных конфликтов для КИМ-3.2 (Часть 1). | Использовать преподавателем для демонстрации на проекторе того, как выглядит конфликт в разных ветках. Показать, как маркеры конфликта (`<<<<<<<`) соотносятся с историей коммитов. |
| **M3 repo3 linear with conflicts.zip** | Обучение идеализации истории (split, move) в КИМ-3.2. | **Важно:** обратить внимание студентов на то, что строки помечены номерами оригинальных коммитов. Это идеальный полигон для демонстрации `git add -p` (разбиение ханков) и интерактивного `rebase`, так как позволяет наглядно отследить судьбу конкретной строки кода. |
| **miniPaint (GitHub)** | Основа для КИМ-3.2 (Часть 2: бисекция). | **Требует предварительной подготовки преподавателя:** 1. Проверить, собирается и запускается ли проект на ОС, установленных в компьютерном классе. 2. Самостоятельно провести `git bisect` и зафиксировать 3–4 конкретных изменения в функциональности (например, появление новой кисти или изменение поведения инструмента) между ПЗК и HEAD. Они могут быть использованы для демонстрации или выданы как индивидуальные задания студентам. Репозиторий можно использовать как образец для определения того, подходит ли репозиторий индустриального партнера для выполнения лабораторной работы. |
