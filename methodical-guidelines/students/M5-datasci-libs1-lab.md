### 4. Часть 1. Генерация и подготовка данных

**Задача:** Создать синтетический датасет для классификации и разбить его на обучающую и тестовую выборки.

**Что нужно сделать самостоятельно:**

**Шаг 1. Генерация датасета**
1. Создайте точки, принадлежащие двум или трем классам.
2. Используйте генератор случайных чисел из NumPy для создания гауссовых облаков со смещением.
3. Каждый класс должен иметь свой центр и разброс.
4. Визуализируйте полученные данные с помощью scatter plot.

**Шаг 2. Разбиение на выборки**
1. Реализуйте функцию `train_test_split` с использованием только NumPy.
2. Функция должна принимать признаки `X`, метки `y`, размер тестовой выборки и параметр случайности.
3. Используйте `np.random.permutation` для перемешивания данных.
4. Верните обучающую и тестовую выборки.

**На что обратить внимание:**
- Какой процент данных должен попасть в тестовую выборку?
- Как обеспечить воспроизводимость результатов?

**Рекомендации:**
- Изучите `np.random.normal` для создания гауссовых облаков.
- Изучите `np.random.permutation` для перемешивания данных.
- Изучите, как работает параметр `random_state`.

### 5. Часть 2. Реализация KNN

**Задача:** Реализовать классификатор k-ближайших соседей с использованием только NumPy.

**Что нужно сделать самостоятельно:**

**Шаг 1. Создание класса**
1. Создайте класс `KNNClassifier`.
2. Реализуйте метод `fit`, который сохраняет обучающие данные.

**Шаг 2. Реализация предсказания**
1. Реализуйте метод `predict`, который принимает тестовые данные и значение `k`.
2. Для каждой точки из тестовой выборки:
   - Вычислите расстояние до всех точек из обучающей выборки.
   - Найдите индексы `k` ближайших соседей.
   - Определите наиболее частый класс среди соседей (голосование).
3. Верните массив предсказанных классов.

**Шаг 3. Реализация оценки качества**
1. Реализуйте метод `score`, который вычисляет точность (accuracy) на тестовых данных.

**На что обратить внимание:**
- Какую метрику расстояния использовать? (Евклидово, Манхэттенское)
- Как эффективно вычислить расстояния без явных циклов? (векторизация)
- Что делать, если несколько классов набрали одинаковое количество голосов?

**Рекомендации:**
- Изучите broadcasting в NumPy для вычисления матрицы расстояний.
- Изучите `np.argsort` или `np.argpartition` для поиска ближайших соседей.
- Изучите `np.bincount` или `np.unique` для голосования.

### 6. Часть 3. Оптимизация вычислений

**Задача:** Сравнить различные способы вычисления матрицы расстояний и выбрать оптимальный.

**Что нужно сделать самостоятельно:**

**Шаг 1. Реализация разных подходов**
1. Реализуйте вычисление матрицы расстояний тремя способами:
   - Вложенные циклы Python (без NumPy).
   - Broadcasting в NumPy.
   - Готовая функция из `scipy.spatial.distance`.
2. Убедитесь, что все три способа дают одинаковый результат.

**Шаг 2. Профилирование производительности**
1. Используйте `%timeit` в Jupyter (или `time` в скрипте) для замера времени выполнения каждого способа.
2. Проведите замеры на датасетах разного размера (например, 100, 500, 1000, 5000 точек).
3. Постройте график зависимости времени выполнения от размера данных.

**Шаг 3. Выводы**
1. Какой способ оказался самым быстрым? Почему?
2. Насколько broadcasting быстрее циклов?
3. Есть ли ситуации, где библиотечная функция (`scipy`) проигрывает broadcasting?

**На что обратить внимание:**
- Какой способ лучше масштабируется с ростом данных?
- Есть ли ограничения у broadcasting по памяти?

**Рекомендации:**
- Изучите `%timeit` в Jupyter Notebook.
- Изучите `scipy.spatial.distance.cdist`.
- Изучите формулу евклидова расстояния и её реализацию через broadcasting.

### 7. Часть 4. Многопоточная обработка

**Задача:** Ускорить вычисления с помощью многопоточности.

**Что нужно сделать самостоятельно:**

**Шаг 1. Разбиение данных**
1. Разбейте тестовую выборку на части (например, по 50–100 точек в каждой).
2. Определите количество частей = количество потоков.

**Шаг 2. Параллельная обработка**
1. Для каждой части выполните предсказание в отдельном потоке.
2. Используйте `ThreadPoolExecutor` для управления потоками.
3. Соберите результаты из всех потоков и объедините их.

**Шаг 3. Сравнение производительности**
1. Замерьте время выполнения многопоточной версии.
2. Замерьте время выполнения однопоточной версии.
3. Сравните результаты и вычислите ускорение.

**На что обратить внимание:**
- Есть ли прирост производительности? На сколько процентов?
- Почему многопоточность может НЕ давать прироста в Python? (GIL)
- Какой размер части оптимален для вашего датасета?

**Рекомендации:**
- Изучите модуль `concurrent.futures.ThreadPoolExecutor`.
- Изучите проблему GIL в Python и её влияние на многопоточные вычисления.
- Изучите альтернативы: `multiprocessing` для CPU-интенсивных задач.

**На что обратить внимание:**
