# Методические рекомендации по выполнению лабораторной работы
## Модуль 2. Базы данных и SQL

### 1. Цель работы
Научиться проектировать схему реляционной базы данных, создавать и наполнять таблицы в PostgreSQL, выполнять сложные аналитические запросы, оптимизировать производительность и интегрировать БД с Python с использованием многопоточности для повышения производительности ETL-процессов.

---

### 2. Общая структура работы

Лабораторная работа состоит из шести частей, каждая из которых проверяет отдельный навык. Выполняйте их последовательно.

| Часть | Название | Что нужно сделать |
| :---: | :--- | :--- |
| 1 | Проектирование схемы БД | Разработать ER-диаграмму на основе датасета Titanic |
| 2 | Создание и наполнение таблиц | Написать SQL-скрипты, создать таблицы, загрузить данные |
| 3 | Аналитические запросы | Написать сложные SQL-запросы для бизнес-аналитики |
| 4 | Оптимизация запросов | Создать индексы, проанализировать производительность |
| 5 | Транзакции | Реализовать операцию с возможностью отката |
| 6 | Интеграция с Python и многопоточность | Реализовать ETL-пайплайн с параллельной обработкой |

---

### 3. Подготовка окружения

#### 3.1. Установка PostgreSQL

**Задача:** Установить PostgreSQL и создать базу данных для работы.

**Что нужно сделать самостоятельно:**
1. Выберите способ установки: локальная установка или через Docker.
2. Установите PostgreSQL, следуя официальной документации.
3. Создайте базу данных с именем `titanic_db`.
4. Убедитесь, что подключение работает через `psql` или любой GUI-клиент (pgAdmin, DBeaver).
5. Запомните параметры подключения (хост, порт, пользователь, пароль) — они понадобятся для Python-скрипта.

**Рекомендуемые инструменты:**
- Официальная документация PostgreSQL: https://www.postgresql.org/docs/
- Docker Hub: https://hub.docker.com/_/postgres
- pgAdmin: https://www.pgadmin.org/

#### 3.2. Установка Python-зависимостей

**Задача:** Настроить окружение для работы с БД из Python.

**Что нужно сделать самостоятельно:**
1. Создайте виртуальное окружение Python.
2. Установите библиотеки для работы с PostgreSQL и анализа данных.
3. Проверьте, что импорт библиотек работает без ошибок.

**Необходимые библиотеки:**
- Для подключения к PostgreSQL из Python (изучите документацию `psycopg2`).
- Для работы с данными (изучите документацию `pandas`).
- Для многопоточности (изучите модуль `concurrent.futures`).

**Рекомендуемые источники:**
- Документация psycopg2: https://www.psycopg.org/docs/
- Документация pandas: https://pandas.pydata.org/docs/
- Документация concurrent.futures: https://docs.python.org/3/library/concurrent.futures.html

#### 3.3. Датасет

**Задача:** Получить датасет Titanic для загрузки в БД.

**Что нужно сделать самостоятельно:**
1. Скачайте датасет Titanic с Kaggle: https://www.kaggle.com/c/titanic/data
2. Или используйте встроенный датасет из библиотеки Seaborn и сохраните его в CSV.
3. Изучите структуру датасета: какие поля есть, какие типы данных.

### 4. Часть 1. Проектирование схемы базы данных

**Задача:** Разработать нормализованную схему БД на основе датасета Titanic.

**Что нужно сделать самостоятельно:**
1. Изучите структуру датасета и определите возможные сущности.
2. Ответьте на вопросы:
   - Какие поля могут быть вынесены в отдельные таблицы?
   - Какие таблицы должны быть связаны внешними ключами?
   - Какие ограничения целостности нужны (NOT NULL, CHECK, UNIQUE)?
3. Примените нормализацию:
   - Приведите данные к 1NF.
   - Приведите к 2NF (устраните частичные зависимости).
   - Приведите к 3NF (устраните транзитивные зависимости).
4. Нарисуйте ER-диаграмму (можно в любом удобном инструменте).
5. Запишите описание таблиц с полями, типами и ключами.

**Рекомендации:**
- Вспомните теорию нормализации из курса баз данных.
- Используйте онлайн-инструменты для создания ER-диаграмм (draw.io, dbdiagram.io).

### 5. Часть 2. Создание и наполнение таблиц

**Задача:** Реализовать схему в PostgreSQL и загрузить данные.

**Что нужно сделать самостоятельно:**
1. Напишите SQL-скрипты:
   - `CREATE TABLE` для каждой сущности с ограничениями.
   - `ALTER TABLE` для добавления внешних ключей (если нужно).
2. Создайте таблицы в PostgreSQL.
3. Загрузите данные из CSV:
   - Изучите команду `COPY` в PostgreSQL для массовой загрузки.
   - Или напишите Python-скрипт для чтения CSV и вставки данных через `psycopg2`.
4. Проверьте, что данные загружены корректно (выполните `SELECT COUNT(*)`).

**На что обратить внимание:**
- Ограничения целостности должны работать: попробуйте вставить некорректные данные и посмотрите на реакцию БД.
- Внешние ключи должны обеспечивать ссылочную целостность.

**Рекомендации:**
- Изучите синтаксис `COPY` в документации PostgreSQL.
- Для Python-скрипта изучите, как выполнять пакетные вставки (`executemany`).

### 6. Часть 3. Аналитические запросы

**Задача:** Написать SQL-запросы для аналитики выживаемости пассажиров.

**Что нужно сделать самостоятельно:**

**Запрос 1. Группировка и агрегация**
- Рассчитайте долю выживших пассажиров в зависимости от пола и класса обслуживания.
- Рассчитайте средний возраст и среднюю стоимость билета для выживших и погибших.

**Запрос 2. Соединение таблиц**
- Выведите полный отчет по пассажирам: имя, возраст, класс, порт посадки, стоимость билета, статус выживания.

**Запрос 3. Оконные функции**
- Пронумеруйте пассажиров по возрасту в каждом классе (от самого старшего к младшему).
- Для каждого класса найдите пассажира с самой высокой стоимостью билета.

**Запрос 4. Подзапросы**
- Найдите пассажиров, чья стоимость билета выше средней стоимости в их классе.
- Найдите классы обслуживания, где доля выживших превышает 50%.

**Рекомендации:**
- Используйте `GROUP BY` и агрегатные функции (`AVG`, `COUNT`, `SUM`).
- Для оконных функций изучите `ROW_NUMBER()`, `RANK()`, `PARTITION BY`.
- Для подзапросов используйте `IN`, `EXISTS`, или вложенные `SELECT`.

### 7. Часть 4. Оптимизация запросов

**Задача:** Ускорить выполнение запросов с помощью индексов.

**Что нужно сделать самостоятельно:**
1. Выполните аналитический запрос (например, выборку выживших) и зафиксируйте время выполнения.
2. Изучите, какие индексы можно создать для ускорения этого запроса.
3. Создайте индексы на нужных столбцах.
4. Снова выполните запрос и сравните время выполнения.
5. Используйте `EXPLAIN ANALYZE`, чтобы увидеть план выполнения запроса до и после создания индексов.

**На что обратить внимание:**
- Как изменился план запроса? (Seq Scan vs Index Scan)
- На сколько процентов ускорился запрос?
- Какие столбцы лучше всего индексировать для аналитических запросов?

**Рекомендации:**
- Изучите синтаксис `CREATE INDEX` в PostgreSQL.
- Изучите работу команды `EXPLAIN ANALYZE`.

---

### 8. Часть 5. Транзакции

**Задача:** Реализовать операцию обновления данных с контролем целостности.

**Что нужно сделать самостоятельно:**
1. Найдите пассажира с некорректным возрастом (или выберите любого для примера).
2. Начните транзакцию (`BEGIN`).
3. Обновите возраст пассажира.
4. Проверьте результат с помощью `SELECT`.
5. Если всё правильно — зафиксируйте изменения (`COMMIT`).
6. Если что-то пошло не так — откатите изменения (`ROLLBACK`).

**На что обратить внимание:**
- Что происходит с данными при `ROLLBACK`?
- Что происходит с данными при `COMMIT`?

**Рекомендации:**
- Изучите свойства ACID транзакций.
- Поэкспериментируйте с разными сценариями (успешное завершение, ошибка в середине транзакции).

### 9. Часть 6. Интеграция с Python и многопоточность

**Задача:** Реализовать ETL-пайплайн с многопоточной загрузкой данных и параллельным выполнением запросов.

**Что нужно сделать самостоятельно:**

**Задание А. Многопоточная загрузка:**
1. Напишите функцию, которая загружает одну "порцию" данных из CSV в БД.
2. Разбейте CSV-файл на части (используйте параметр `chunksize` в `pandas.read_csv`).
3. Используйте `ThreadPoolExecutor`, чтобы запустить загрузку каждой части в отдельном потоке.
4. Замерьте время выполнения многопоточной загрузки.
5. Реализуйте однопоточную версию загрузки и замерьте её время.
6. Сравните результаты и сделайте выводы.

**Задание Б. Многопоточная выгрузка:**
1. Напишите функцию, которая выполняет один SQL-запрос и возвращает результат.
2. Подготовьте 3–4 различных аналитических запроса.
3. Используйте `ThreadPoolExecutor`, чтобы выполнить их параллельно.
4. Замерьте время выполнения многопоточной выгрузки.
5. Выполните те же запросы последовательно и замерьте время.
6. Сравните результаты и сделайте выводы.

**На что обратить внимание:**
- Какое соединение с БД используется в каждом потоке? (Нужно ли создавать отдельное соединение для каждого потока?)
- Какой прирост производительности дает многопоточность?
- Есть ли ограничения у многопоточного подхода в Python (GIL)?

**Рекомендации:**
- Изучите, как `psycopg2` работает с многопоточностью.
- Изучите пул соединений (connection pool) для оптимизации.
- Изучите модуль `time` для замера производительности.

### 10. Оформление отчета

Отчет должен содержать:

1. **Титульный лист** — название работы, ФИО, группа.
2. **Цель работы** — сформулируйте самостоятельно.
3. **ER-диаграмма** — скриншот или рисунок вашей схемы.
4. **SQL-скрипты** — полный код создания таблиц и загрузки данных.
5. **Результаты запросов** — текст запросов + скриншоты результатов.
6. **Оптимизация** — сравнение времени выполнения до и после создания индексов (таблица, график).
7. **Транзакции** — пример `BEGIN/COMMIT/ROLLBACK` с пояснением.
8. **Многопоточность** — код Python-скрипта + таблица сравнения производительности.
9. **Выводы** — по каждой части работы (что нового узнали, с какими проблемами столкнулись).
