# Доклад. Проектирование архитектуры ИИ-приложений

## Формат
Устный доклад 5–7 минут на практическом занятии.

## Список тем

1. **Архитектурные паттерны ИИ-приложений: монолит, сервисы, микросервисы и гибридные подходы**  
   Доклад должен сравнить четыре типа архитектур с точки зрения применимости к ML-системам. Следует разобрать, когда монолит удобен (быстрый прототип, небольшой проект), а когда требуется переход к сервисной или микросервисной структуре (разделение предобработки, инференса, мониторинга). Важно показать гибридные схемы, где часть компонентов работает как микросервисы, а часть — как модули монолита. В заключение стоит привести критерии выбора для реального проекта (размер команды, нагрузка, требования к масштабированию) и пример гибридной архитектуры для сервиса распознавания дорожных знаков.

2. **Организация слоя данных в ИИ-системах: хранилища, пайплайны, аугментация и версионирование**  
   Доклад должен охватить жизненный цикл данных от сбора до подачи в модель. Следует сравнить типы хранилищ (объектное S3, реляционные БД, колоночные ClickHouse, векторные базы) и их роль в обучении и инференсе. Важно показать построение пайплайнов предобработки с использованием библиотек (Pandas, Dask, Albumentations, torchvision) и обсудить аугментацию как способ увеличения выборки. Также стоит затронуть версионирование данных (DVC) и обеспечение воспроизводимости экспериментов. В заключение — привести схему пайплайна для датасета дорожных знаков (загрузка, очистка, аугментация, запись в S3).

3. **Обеспечение масштабируемости ИИ-приложений: горизонтальное и вертикальное масштабирование, автоматическое масштабирование в Kubernetes**  
   Доклад должен раскрыть, как адаптировать систему под растущую нагрузку. Следует сравнить вертикальное (увеличение ресурсов одной ноды) и горизонтальное (добавление новых реплик) масштабирование, показав их плюсы и ограничения для ML-задач (например, GPU-инференс лучше масштабируется горизонтально). Важно обсудить Horizontal Pod Autoscaler в Kubernetes, настройку метрик (CPU, память, пользовательские метрики) и использование очередей сообщений для асинхронной обработки. В заключение — привести пример конфигурации HPA для сервиса распознавания дорожных знаков.

4. **Выбор технологического стека для компьютерного зрения в ИИ-архитектурах**  
   Доклад должен охватить фреймворки (PyTorch, TensorFlow), библиотеки (OpenCV, Albumentations), популярные модели (YOLO, MMDetection) и аппаратное ускорение (GPU NVIDIA, TPU, нейропроцессоры). Следует сравнить производительность, удобство разработки и поддержку разных форматов. Важно обсудить, как выбор влияет на инфраструктуру: например, использование ONNX для кроссплатформенного инференса. В заключение — привести рекомендации для задачи распознавания дорожных знаков (например, ResNet-18 на PyTorch с аугментациями).

5. **Платформа Yandex Cloud для ИИ: основные сущности и интеграция сервисов**  
   Доклад должен дать обзор ключевых сервисов, необходимых для построения ML-решений. Следует рассмотреть Compute Cloud (виртуальные машины с GPU), Managed Kubernetes, Object Storage, Managed Databases (PostgreSQL, ClickHouse), а также AI-сервисы: Yandex Vision, SpeechKit, Translate. Важно показать, как они взаимодействуют через IAM, VPC, Security Groups и как использовать сервисные аккаунты для авторизации. В заключение — привести сквозной пример архитектуры: загрузка данных в Object Storage → обучение в DataSphere → развертывание через Kubernetes → мониторинг в DataLens.

## Требования к содержанию доклада
- Длительность — 5–7 минут.
- Обязательное использование визуальных материалов (презентация, схемы архитектуры, диаграммы компонентов).
- Ссылки на реальные примеры из практики (конкретные инструменты, библиотеки, сервисы).
- Чёткая структура: введение → основная часть с анализом и примерами → выводы и рекомендации.
- Ответы на вопросы аудитории после выступления.

## Требования к сдаче
- Презентация (3–5 слайдов) в формате PDF или PowerPoint.
- Тезисы доклада (краткий конспект) — 1–2 страницы.
