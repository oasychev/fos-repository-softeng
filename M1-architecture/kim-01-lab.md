# Лабораторная работа №1. Основы ML-инфраструктуры: от контейнера до облака

## Цель работы
Освоить навыки контейнеризации и развёртывания ML-сервисов в облачной среде. Создать и развернуть в Yandex Cloud контейнеризированный веб-сервис распознавания дорожных знаков, настроить базовую инфраструктуру и CI/CD пайплайн.

## Задание

### Часть 1. Разработка и контейнеризация сервиса
1. Разработать веб-сервис на FastAPI, который:
   - принимает изображение дорожного знака (через HTTP‑запрос);
   - возвращает предсказанный класс знака (например, «Stop», «Speed limit 60» и т.д.).
2. Использовать предварительно обученную модель (например, ResNet‑18 или YOLO, обученную на датасете дорожных знаков, например, GTSRB).
3. Упаковать сервис в Docker-образ:
   - написать Dockerfile с оптимизацией размера (многостадийная сборка, использование slim-образов);
   - обеспечить передачу модели и зависимостей.
4. Проверить локальный запуск контейнера и работоспособность API.

### Часть 2. Развёртывание в облаке и настройка CI/CD
1. Создать виртуальную машину в Yandex Compute Cloud (или использовать Managed Kubernetes) с необходимыми ресурсами (CPU, RAM, GPU – по возможности).
2. Настроить сетевые правила (Security Groups) для доступа к сервису по порту 80/443.
3. Развернуть контейнер на созданной инфраструктуре (docker run или через Kubernetes Deployment).
4. Настроить CI/CD пайплайн (GitHub Actions или GitLab CI) для автоматической сборки образа при пуше в репозиторий и деплоя на сервер.
5. Обеспечить логирование (логи контейнера, структурированное логирование) и базовый мониторинг (проверка доступности эндпоинта).

### Часть 3. Документирование
1. Подготовить README.md с инструкцией по сборке, развертыванию и тестированию сервиса.
2. Описать архитектуру решения (схема компонентов: клиент → API Gateway/Load Balancer → контейнер → модель).

## Требования к сдаче
- Репозиторий с кодом (FastAPI‑приложение, Dockerfile, скрипты для CI/CD).
- Ссылка на работающий сервис в Yandex Cloud (публичный IP или домен).
- README.md с инструкцией по развертыванию.
- Презентация (3–5 слайдов) с описанием архитектуры, принятых решений и результатов.
