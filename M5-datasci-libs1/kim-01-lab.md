# Лабораторная работа. Введение в Data Science. NumPy

## Цель работы
Научиться применять библиотеку NumPy для реализации базовых алгоритмов машинного обучения с нуля, используя векторизованные вычисления и оптимизацию производительности.

## Задание

Реализовать классификатор на основе метода k-ближайших соседей (k-NN) с использованием только NumPy.

### Часть 1. Генерация и подготовка данных

1. Сгенерировать синтетический датасет для классификации:
   - Использовать `np.random` для создания точек, принадлежащих двум или трем классам (гауссовы облака).
2. Реализовать функцию `train_test_split`:
   - Перемешать данные с использованием `np.random.permutation`.
   - Разбить на обучающую и тестовую выборки (test_size=0.2).

### Часть 2. Реализация KNN

Реализовать класс `KNNClassifier`:

1. `fit(X_train, y_train)` — сохраняет обучающие данные.
2. `predict(X_test, k=3)`:
   - Вычислить матрицу расстояний с использованием broadcasting.
   - Найти k ближайших соседей (`np.argsort`).
   - Выполнить голосование (`np.bincount`).
3. `score(X_test, y_test)` — вычисляет точность (accuracy).

### Часть 3. Оптимизация вычислений

1. Сравнить различные методы вычисления матрицы расстояний:
   - Вложенные циклы Python.
   - Broadcasting.
   - `scipy.spatial.distance.cdist`.
2. Провести профилирование производительности с `%timeit`.
3. Сделать выводы о выборе оптимального метода.

### Часть 4. Многопоточная обработка

1. Разбить данные на части.
2. Обработать части параллельно с использованием `threading` или `concurrent.futures`.
3. Сравнить время выполнения с однопоточной версией.

### Часть 5. Сравнение с Scikit-learn

1. Обучить модель KNN из Scikit-learn на тех же данных.
2. Сравнить точность и скорость работы с собственной реализацией.
3. Проанализировать преимущества библиотечной реализации.

### Часть 6. Работа с большими данными (дополнительно)

1. Использовать `np.memmap` для работы с массивом, не помещающимся в ОЗУ.
2. Реализовать потоковую обработку данных по частям.
