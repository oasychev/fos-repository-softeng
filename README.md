# Фонд оценочных средств по дисциплине «Разработка прикладных программных решений для систем искусственного интеллекта»
Этот репозиторий предназначен для открытого и воспроизводимого представления фонда оценочных средств (ФОС) по дисциплине «Разработка прикладных программных решений для систем искусственного интеллекта». Здесь объединены рабочая программа дисциплины, модель измерения результатов обучения, контрольно-измерительные материалы (КИМ), критерии оценивания, методические рекомендации и используемые образовательные ресурсы. Результаты обучения связываются с компетентностно-ролевой моделью в области искусственного интеллекта (КРМ версии 3.0).

## 1. О дисциплине

**Цель дисциплины.** Формирование системных знаний и практических навыков разработки промышленных программных решений для систем искусственного интеллекта с использованием Python, классических ML-методов и инженерных практик MLOps.

**Основные задачи.**
1. Научиться проектировать промышленные программные решения (бэкенд-разработка) на основе Python-стека и SQL баз данных.
2. Развить умения и навыки промышленной разработки программных решений (Git, GitFlow, GitMerge, poetry, pytest, pydantic, PEP8).
3. Приобрести навыки реализации ML-моделей на основе библиотек и их включения в программную систему ( NumPy, Pandas, Matplotlib, Seaborn, Scikit-learn, ).
4. Изучить методы и библиотеки разработки внешних интерфейсов системы и ее развертывания в продукционной среде (REST API, FastAPI, Swagger, Docker, DockerHub, CI/CD).

**Структура и содержание.** Дисциплина состоит из 8 модулей: проектирование архитектуры ИИ-приложений, базы данных и SQL, использование контроля версий при разработке группового проекта, инструменты промышленной разработки на Python, библиотеки для работы с данными, анализ данных с Pandas и визуализация с Matplotlib, введение в веб-разработку FastAPI, контейнеризация и деплой Docker. Обучение включает лекции, лабораторные работы, курсовую работу и самостоятельную работу и завершается защитой курсовой работы. Зачет выставляется на основе результатов выполнения КИМ по модулям и курсовой работы.

**Пререквизиты и связи с другими дисциплинами.** 
Для успешного освоения дисциплины студент должен владеть следующими компетенциями и индикаторами:

- PL-1.1 (Б): базовый Python (синтаксис, структуры данных, ООП) — «Введение в ИТ»
- PL-1.2 (Б): выбор инструментов Python для обработки данных — «Введение в ИТ»
- BD-1.2 (Б): методы предварительного анализа данных, визуализация — «Введение в ИТ»
- BD-2.1 (Б): требования к наборам и качеству данных для МО — «Введение в ИТ»
- BD-2.2 (Б): сбор, проверка, очистка данных — «Введение в ИТ»
- BD-3.1 (Б): создание БД в реляционных СУБД, SQL-запросы — смежный курс «БД»
- ML-3.1 (Б): описание базовых моделей МО — «Машинное обучение»
- LC-5.1 (Б): основы Git — «Введение в ИТ»
- LC-5.2 (Б): основы версионирования данных — «Машинное обучение»
- MF-1.2 (Б): анализ задач искусственного интеллекта — «Математические основы ИИ»
- MF-4.1 (Б): решение задач анализа данных для МО — «Математические основы ИИ»

**Планируемые результаты обучения.** По завершении дисциплины обучающийся сможет:
1. Проектировать архитектуру ИИ-приложений и выбирать стек технологий
2. Разрабатывать ETL-пайплайны и оптимизировать SQL-запросы в PostgreSQL, обеспечивая многопоточную загрузку данных из Python.
3. Применять продвинутые операции Git (rebase, rebase -i, bisect, cherry-pick) и разрешать конфликты слияния в командной разработке.
4. Обрабатывать большие наборы данных с использованием векторизованных операций NumPy и оптимизирует память в Pandas (chunking, dtype, usecols).
5. Разрабатывать REST API на FastAPI с асинхронными эндпоинтами, валидацией данных (Pydantic) и автоматической документацией (Swagger).
6. Упаковывать приложение в Docker-контейнер, описывать многоконтейнерное окружение через Docker Compose и настраивать базовый CI/CD пайплайн (GitHub Actions).

Полное описание приведено в [РПД](docs/rpd.md).

## 4. Итоговая оценка

**Текущий контроль:** каждое устное сообщение — 2 балла; каждая лабораторная работа — до 7 баллов (всего 8 лабораторных работ); групповая игра модуля 3 - 4 балла.  
**Промежуточная аттестация:** курсовая работа — 40 баллов.
**Итоговая оценка:**  - зачет — набрать более 60 баллов
