# Лабораторная работа. Базы данных и SQL

## Цель работы
Научиться проектировать схему реляционной базы данных, создавать и наполнять таблицы в PostgreSQL, выполнять сложные аналитические запросы, оптимизировать производительность, работать с транзакциями и интегрировать БД с Python с использованием многопоточности для повышения производительности.

## Задание

### Часть 1. Проектирование схемы базы данных

На основе датасета Titanic спроектировать схему базы данных:

1. Определить сущности: **Passengers**, **Tickets**, **Classes**, **Embarkments**.
2. Определить атрибуты для каждой сущности.
3. Определить первичные и внешние ключи.
4. Обосновать выбор схемы (акцент на удобство аналитических запросов).

### Часть 2. Создание и наполнение таблиц

1. Установить PostgreSQL (локально или через Docker).
2. Написать SQL-скрипты для создания таблиц с ограничениями (`NOT NULL`, `CHECK`, `FOREIGN KEY`).
3. Загрузить данные из CSV-файла в таблицы:
   - Использовать команду `COPY` в PostgreSQL для массовой загрузки.
   - Или написать Python-скрипт с использованием `psycopg2` для вставки данных.

### Часть 3. Аналитические запросы

Выполнить запросы для ответа на бизнес-вопросы:

1. **Группировка и агрегация:**
   - Рассчитать долю выживших по полу и классу обслуживания.
   - Рассчитать средний возраст и среднюю стоимость билета для выживших и погибших.
2. **Соединение таблиц:**
   - Вывести полный отчет по пассажирам: имя, возраст, класс, порт посадки, стоимость билета, статус выживания.
3. **Оконные функции:**
   - Пронумеровать пассажиров по возрасту в каждом классе (`ROW_NUMBER`).
   - Для каждого класса показать пассажира с самой высокой стоимостью билета (`RANK`).
4. **Подзапросы:**
   - Найти пассажиров, чья стоимость билета выше средней в своем классе.
   - Найти классы обслуживания, где доля выживших выше 50%.

### Часть 4. Оптимизация запросов

1. Создать индексы для ускорения запросов:
   - `CREATE INDEX idx_survived ON Passengers(Survived);`
   - `CREATE INDEX idx_pclass ON Passengers(Pclass);`
2. Использовать `EXPLAIN ANALYZE` для сравнения производительности запросов до и после создания индексов.

### Часть 5. Транзакции

1. Начать транзакцию (`BEGIN`).
2. Обновить информацию о пассажире (например, исправить возраст).
3. Проверить результат (`SELECT`).
4. Зафиксировать (`COMMIT`) или откатить (`ROLLBACK`) изменения.

### Часть 6. Интеграция с Python и многопоточность

Написать Python-скрипт с использованием `psycopg2` и `ThreadPoolExecutor`:

1. Подключиться к базе данных PostgreSQL (использовать пул соединений).
2. Реализовать многопоточную загрузку данных:
   - Разбить CSV-файл на части.
   - В каждом потоке загрузить свою часть данных в БД с использованием `COPY` или пакетной вставки.
   - Сравнить время выполнения с однопоточной версией.
3. Реализовать многопоточную выгрузку данных:
   - Выполнить несколько аналитических запросов параллельно в разных потоках.
   - Собрать результаты и объединить их.
   - Сравнить время выполнения с последовательной версией.
4. Написать отчет о сравнении производительности однопоточной и многопоточной версий.

## Требования к сдаче

Результат работы должен включать:

1. SQL-скрипты (создание таблиц, наполнение, аналитические запросы).
2. Python-скрипт с реализацией многопоточной загрузки и выгрузки данных.
3. Отчет с результатами выполнения запросов, анализа производительности и сравнения однопоточного/многопоточного подходов.
