# КИМ-2.1. Устные доклады

> Устный доклад проверяет и развивает умения студента собирать и систематизировать профессиональную информацию из различных источников. На доклад отводится 5 минут на лекционном или практическом занятии, после чего следует обсуждение с преподавателем и группой, где участники могут задавать вопросы, дополнять и корректировать информацию из сообщения. При сборе информации студент может использовать любые методы включая ИИ-ассисстентов, но ответственность за полноту и точность собранной информации ложится на самого докладчика.

Устные доклады модуля 7 фокусируются на лучших практиках и применении пройденных технологий и инструментов: Docker, DockerHub, Docker Compose, Github Actions.

## Критерии оценки

| Критерии | Баллы |
| -------- | :---: |
| Доклад не был подготовлен, или содержание доклада критически отличается от заявленной темы | 0 |
| Доклад содержит много недоточностей, информация недостоверна | 1 |
| Докладчик не смог ответить на вопросы преподавателя | 1 |
| Доклад широко ракрывает тему и практическое аспекты, ответы на вопросы даны в полной мере | 2 |

## Тайминги

|  Критерии  |  Время   |
|------------|:--------:|
| Доклад     | 5 минут  |
| Обсуждение | 10 минут |

## Темы докладов

Студент может выбрать тему из предложенных или согласовать свою тему с преподавателем.

Рекоммендуемые темы докладов:

1. Принципы контейнеризации современных программных систем
2. Контейнеризация и виртуализация: выбор технологии для различных задач
3. Лучшие практики создания Docker-образов
4. Лучшие практики организации Docker-проектов
5. Управление зависимостями приложений с помощью Docker
6. Стратегии версионирования Docker-образов
7. Выбор и использование базовых Docker-образов
8. Хранение данных в контейнеризированных приложениях
9. Организация взаимодействия контейнеров в многосервисных приложениях
10. Docker Compose как инструмент описания инфраструктуры приложения
11. Типичные ошибки и антипаттерны при использовании Docker
12. Безопасность контейнеризированных приложений
13. Публикация и распространение Docker-образов через Docker Hub
14. Воспроизводимость программных окружений с помощью Docker
15. Контейнеризация в проектах машинного обучения
16. Docker как инструмент командной разработки
17. Роль Docker в современных DevOps-практиках
18. CI/CD как средство автоматизации жизненного цикла программного обеспечения
19. GitHub Actions: возможности и типовые сценарии применения
20. Автоматизация сборки, тестирования и развертывания приложений
21. Контейнеризация в облачной инфраструктуре
22. Ограничения контейнеризации и случаи ее нецелесообразного применения
23. Современные тенденции развития технологий контейнеризации
