# КИМ-2.1. Устные доклады

>| Устный доклад проверяет и развивает умения студента собирать и систематизировать профессиональную информацию из различных источников. На доклад отводится 5 минут на лекционном или практическом занятии, после чего следует обсуждение с преподавателем и группой, где участники могут задавать вопросы, дополнять и корректировать информацию из сообщения. При сборе информации студент может использовать любые методы включая ИИ-ассисстентов, но ответственность за полноту и точность собранной информации ложится на самого докладчика.

Устные доклады модуля 7 фокусируются на лучших практиках работы и применении пройденных инструментов: HTTP, JSON, RestAPI, Swaggeer, FastAPI. 

## Критерии оценки

| Критерии | Баллы |
|----------|:-----:|
| Доклад не был подготовлен, или содержание доклада критически отличается от заявленной темы | 0 |
| Доклад содержит много недоточностей, информация недостоверна | 1 |
| Докладчик не смог ответить на вопросы преподавателя | 1 |
| Доклад широко ракрывает тему и практическое аспекты, ответы на вопросы даны в полной мере | 2 |

## Тайминги

|  Критерии  |  Время   |
|------------|:--------:|
| Доклад     | 5 минут  |
| Обсуждение | 10 минут |

## Темы докладов

Студент может выбрать тему из предложенных или согласовать свою тему с преподавателем.

Рекоммендуемые темы докладов:

1. Принципы проектирования качественного REST API
2. Лучшие практики проектирования URL и эндпойнтов
3. Выбор HTTP-методов при проектировании API
4. Использование HTTP-статус-кодов: рекомендации и типичные ошибки
5. REST API и GraphQL: сравнительный обзор
6. Версионирование REST API
7. OpenAPI: документирование API как часть процесса разработки программного обеспечения
8. Swagger и другие инструменты документирования API: сравнительный обзор
9. Валидация данных в веб-приложениях: назначение и основные подходы
10. Принципы обработки ошибок в REST API
11. Единообразие ответов API и стандартизация форматов данных
12. JSON как стандарт обмена данными между клиентом и сервером
13. Когда асинхронность действительно необходима в веб-приложениях
14. FastAPI среди современных Python-фреймворков для разработки API
15. Критерии выбора веб-фреймворка для Python-проектов
16. Архитектура современных backend-приложений
17. Принципы организации структуры backend-проекта
18. Разделение ответственности в серверных приложениях
19. Middleware: назначение и области применения
20. Основные требования к безопасности REST API
21. Производительность веб-API и факторы, влияющие на нее
22. Масштабируемость REST API
23. Тестирование REST API: основные подходы и инструменты
