# КИМ-7.2. Лабораторное занятие

## Цель работы

Освоить разработку и сопровождение REST API с использованием фреймворка FastAPI, изучить принципы взаимодействия клиента и сервера по протоколу HTTP, научиться использовать средства автоматического документирования API, выполнять анализ, отладку и расширение существующего серверного приложения.

## Формируемые результаты обучения

В результате выполнения лабораторной работы обучающийся должен уметь:

- развертывать серверное приложение на базе FastAPI в локальной среде;
- использовать Swagger (OpenAPI) для исследования и тестирования REST API;
- анализировать структуру HTTP-запросов и ответов;
- выявлять и локализовывать ошибки в работе REST API;
- реализовывать новые REST-эндпойнты с использованием средств FastAPI;
- применять средства валидации входных данных и документирования API.

## Исходные данные

Обучающемуся предоставляется исходный код серверного приложения на базе FastAPI, база данных, документация Swagger (OpenAPI) и индивидуальный вариант задания на реализацию дополнительного функционала.


## Задание

### Часть 1. Исследование и отладка существующего REST API

1. Выполнить настройку локальной среды разработки, развернуть предоставленное серверное приложение (сервер приложений, база данных, Swagger/OpenAPI) и убедиться в его работоспособности.

2. Открыть страницу Swagger (OpenAPI) и исследовать предоставленный REST API:

   * ознакомиться со структурой доступных эндпойнтов;
   * изучить параметры запросов и модели данных;
   * определить поддерживаемые HTTP-методы и возможные HTTP-статус-коды.

3. Выполнить тестирование предоставленных эндпойнтов средствами Swagger:

   * выполнить запросы различных типов (GET, POST, PUT, PATCH, DELETE);
   * сравнить результаты выполнения корректных и некорректных запросов;
   * задокументировать полученные результаты.

4. Исследовать механизм аутентификации API (при наличии), выполнить запросы без авторизации и с использованием предоставленных учетных данных, проанализировать различия в поведении сервиса.

5. Активировать предоставленный преподавателем функциональный модуль (эндпойнт) и выполнить его тестирование средствами Swagger.

6. Проанализировать журналы работы приложения, программный код обработчика запросов, модели данных и структуру базы данных для определения причины возникновения ошибки.

7. Оформить обнаруженный дефект в формате баг-репорта, содержащего описание проблемы, шаги воспроизведения, ожидаемый и фактический результат, а также предполагаемую причину возникновения ошибки.

8. Исправить выявленные ошибки в реализации эндпойнта, выполнить повторное тестирование, убедиться в корректности возвращаемых данных и соответствии результатов содержимому базы данных.

### Часть 2. Расширение функциональности REST API

1. Проанализировать существующую архитектуру проекта и реализацию аналогичных эндпойнтов, определить требования к новому функционалу в соответствии с индивидуальным вариантом задания.

2. Реализовать новые REST-эндпойнты с использованием FastAPI:

   * определить маршруты и соответствующие HTTP-методы;
   * реализовать обработчики запросов;
   * описать модели запросов и ответов средствами Pydantic;
   * обеспечить взаимодействие с базой данных.

3. При необходимости реализовать обработку параметров пути, параметров запроса и тела запроса, а также предусмотреть обработку возможных исключений с использованием средств FastAPI.

4. Обновить автоматически формируемую документацию Swagger (OpenAPI), проверить корректность отображения новых эндпойнтов, моделей данных и параметров запросов.

5. Выполнить тестирование разработанных эндпойнтов средствами Swagger, проверить корректность выполнения основных сценариев использования и обработки ошибочных запросов.

6. Подготовить отчет о выполнении лабораторной работы, включающий результаты тестирования REST API, баг-репорт, описание внесенных изменений и подтверждение корректной работы реализованного функционала.


## Критерии оценивания

**Максимальная оценка за выполнение лабораторной работы составляет 7 баллов.** 
Итоговая оценка определяется на основании суммарного процента выполнения всех этапов лабораторной работы с последующим пропорциональным переводом в балльную систему (7 баллов соответствует 100% выполнения).

Оценка выставляется по результатам выполнения основных этапов лабораторной работы в соответствии с критериями, представленными в таблице.

| Этап лабораторной работы                     | Критерии выполнения                                                                                                                                                                                  | Максимальная доля оценки |
| -------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | -----------------------: |
| **1. Развертывание и исследование REST API** | Серверное приложение успешно развернуто, REST API доступен через Swagger (OpenAPI), выполнено исследование предоставленных эндпойнтов, результаты тестирования задокументированы.                    |                  **20%** |
| **2. Поиск и устранение ошибок**             | Выполнено воспроизведение ошибки, определен ее источник, подготовлен баг-репорт, внесены исправления, подтверждена корректная работа исправленного эндпойнта.                                        |                  **40%** |
| **3. Реализация новых REST-эндпойнтов**      | Реализованы эндпойнты в соответствии с заданием, обеспечена корректная работа маршрутов, моделей данных и взаимодействия с базой данных, документация Swagger обновлена.                             |                  **30%** |

> **Примечание.** Каждый этап лабораторной работы оценивается независимо. Итоговая оценка определяется суммой набранных процентов по всем этапам с последующим переводом в баллы.
